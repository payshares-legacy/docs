Software

Payshares is open source. Here's an overview of our current projects:

  • paysharesd: The network is comprised of peers running paysharesds. It validates the ledger and introduces transactions onto the network.
  • Payshares Client: A reference client implementation. It provides a user friendly interface for interacting with paysharesd. You can see an example of it running here.
  • payshares-lib: A JavaScript API for interacting with paysharesd.
  • Payshares Wallet: A safe web-based store of your secret key. The wallet stores your secret key encrypted with a username and password, which are never actually sent to the wallet. This allows users to store and retrieve their credentials without exposing them to a third party.
  • Payshares Viewer: A viewer of the status of any Payshares account. You can see it in action here.
  • Payshares Documentation: The Payshares documentation repository that anyone can contribute to.

Communicating with the network

There are 3 ways to issue commands to paysharesd.

  • Command line- This is the usual method when you are running paysharesd locally.
  • RPC- paysharesd can be configured to listen for http requests on an arbitrary port. You can then make normal PUT requests with the command and parameters sent as a JSON string in the data.
  • Websocket- paysharesd can also be configured to create a websocket. It is useful if you want to subscribe to events from paysharesd. For example if you want to know when a particular account gets a transaction. All websocket commands can take an optional id integer parameter. This id will be reflected in paysharesd's response.
Payshares Foundation runs two instances of paysharesd you can use if you don't want to run your own.

amount

Many of the API commands accept an argument of amount type. An amount is either an amount of paysharess (XPS) or an amount of some other currency.

An amount of paysharess is always represented in micropaysharess (the smallest divisible payshares unit).

example

55 paysharess:
  "amount" : "55000000"

An amount that represents a currency credit is a triplet of the following parameters:

  • value: the amount of credit.
  • currency: the 3 letter currency code, USD, BTC, EUR etc.
  • issuer: the Payshares account that issued the credit.

example

34.5 BTC:
  "amount" : {
    "value" : "34.5"
    "currency" : "BTC",
    "issuer" : "ganVp9o5emfzpwrG5QVUXqMv8AgLcdvySb",
  }

Quoting numbers

When directly submitting transactions to paysharesd, make sure to quote the numbers. Normally, numbers would not be quotd in JSON so that they are interpreted as a number. However, when used to sign transactions, paysharesd is unable to parse JSON integers greater than 4,294,967,295 (2^32-1). This can cause transactions (such as one with more than 4294.967295 XPS being sent) to be rejected by the paysharesd you are submitting to. Fortunately, paysharesd accepts quoted numbers and correctly interprets them.
